하이픈(-) 목록작성 - 목록의 각 항목을 표시하는데 사용
따옴표{"") - 특정 단어나 문구를 강조하거나, 모델이 정확하게 인식해야 하는 텍스트를 표시할 떄 사용
콜론(:) - 목록이나 설명을 시작할 때 사용
별표(*) - 목록을 나타내거나 특정 내용을 강조할 때 사용
대괄호([]) - 추가 정보나 선택 사항을 나타낼 때 사용
중괄호({}) - 특정조건을 명시할 때 사용
세 개의 백틱을(''') 앞 뒤로 사용 - 코드 블록을 표시하는데 사용, 프롬포트 내에서 코드나 명령어를 명확하게 구분
해시 기호(#) - 마크다운 문법에서 제목이나 섹션을 구분하는 데 사용, 프롬포트를 구조화 하여 모델이 각 부분을 명확하게 이해하도록 도움

대화 질문 설계 요소 : 키워드, 제한, 어조, 부정
키워드와 제한
-핵심 키워드를 정확하게 사용하는 것이 중요
-답변의 범위나 형식을 제한하는 것도 유용(조건을 추가하기)
어조와 부정
-사용자가 요청하면 AI가 더 부드러운 어조로 답변할 가능성 높음
-부정적 표현 사용시 원치 않는 결과를 제외(뭐를 뺴고 알려줘)

퓨샷 러닝과 퓨샷 프롬포팅
퓨샷 러닝(FSL:Few-shot learnig)
- AI 모델이 제한된 수의 예시만으로 새로운 개념이나 작업을 학습하는 기술(적은 데이터로 높은 학습 효율을 보이는 놀라운 능력)
- 기존의 머신러닝모델 : 방대한 양의 데이터, 퓨샷 러닝 : 몇장의 고양이사진 -> 고양이 인식(데이터 부족, 빠른적응필요, 개인화 상황에서 쓰임)
퓨샷 프롬포팅
-퓨샷 러닝의 능력을 극대화, 우리가 원하는 결과를 더 정확하게.

제로샷 프롬포팅 - AI에게 아무런 예시 없이 바로 원하는 결과 요구 -> 방대한 지식 바탕으로 질문에 답함
ex)－다음 문장의 감정을 분류해주세요: '오늘 옷을 사서 돈을 많이 지출했어요 (https://url.kr/uwneqv)

원샷과 투샷 프롬포팅 - 각각 하나 또는 두개의 예시 제공, 원하는 결과 유도, 제로샷보다 정확도가 높고, 간단한 패턴을 학습하는데 효과적 (https://url.kr/uwneqv)

퓨샷 프롬포팅
-여러 개의 예시 제공
-복잡한 패턴이나 규칙을 학습하고 다양한 맥락을 이해하는데 효과적
